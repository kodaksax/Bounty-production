## üîß Technical Details

### Why Synchronous > Asynchronous for This Use Case

**Problem with Async:**
```typescript
// Thread 1: Write operation
await SecureStore.setItemAsync('key', 'value'); // Takes 10-50ms
                                                  ‚Üì
                                                  (Still writing...)
                                                  ‚Üì
// Thread 2: Read operation (starts before write completes)
const value = await SecureStore.getItemAsync('key'); // Might read old value!
```

**Solution with Sync Cache:**
```typescript
// Memory write (synchronous)
inMemoryCache = 'value'; // Instant (in-memory, synchronous)
                         ‚Üì
                         (Cache updated immediately)
                         ‚Üì
// Background persistence (async, but cache already updated)
await SecureStore.setItemAsync('key', 'value'); // 10-50ms (runs separately)

// Read from cache (synchronous)
const value = inMemoryCache; // Instant, always correct!
```

### JavaScript Memory Model

- **Single-threaded**: JavaScript execution is single-threaded
- **Atomic assignment**: `inMemoryCache = value` is atomic
- **No race conditions**: Within JavaScript, synchronous operations don't race
- **Memory lifecycle**: Cache exists for app process lifetime
- **Cleared on**: Force quit, app restart, or explicit clear

KEY FEATURES BY SCREEN
======================

AdminDashboard (/)
- Metrics: bounties (by status), users, escrow volume, transactions
- Quick access cards to main sections
- Pull-to-refresh

AdminBountiesList (/bounties)
- Filter by status: all, open, in_progress, completed, archived
- Shows flagged items with warning banner
- Tap to view detail

AdminBountyDetail (/bounty/[id])
- Full bounty information
- Status transition buttons with confirmation
- Optimistic UI updates

AdminUsersList (/users)
- Filter by status: all, active, suspended, banned
- Shows verification badges
- Stats preview: posted, completed, balance

AdminUserDetail (/user/[id])
- Activity stats grid
- Financial summary
- Account information

AdminTransactionsList (/transactions)
- Filter by type: escrow, release, refund, deposit, withdrawal
- Read-only view with status badges
- Shows related bounty/user IDs

FLOW DIAGRAM
============

1. User enables Admin Mode in Settings (dev toggle)
   ↓
2. Admin tab appears in BottomNav (replaces Profile)
   ↓
3. Tap admin tab → navigates to /(admin)/
   ↓
4. AdminLayout checks isAdmin via context
   ↓
5. If not admin → Redirect to /tabs/bounty-app
   If admin → Show AdminDashboard
   ↓
6. From dashboard, access:
   - Bounties → List → Detail (with actions)
   - Users → List → Detail
   - Transactions → List (read-only)

DATA FLOW
=========

Screen Component
    ↓
useAdminX hook (e.g., useAdminBounties)
    ↓
adminDataClient method (e.g., fetchAdminBounties)
    ↓
Mock data + simulated network delay
    ↓
Returns data to hook
    ↓
Hook updates state (data, loading, error)
    ↓
Screen renders with data / loading / error UI

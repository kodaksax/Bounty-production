### iOS Screenshots (Required Sizes)

#### 6.5" Display (iPhone 14 Pro Max, 15 Pro Max)
- **Resolution:** 1290 x 2796 pixels (portrait) or 2796 x 1290 pixels (landscape)
- **Format:** PNG or JPEG
- **Required:** Minimum 3 screenshots, maximum 10
- **Recommended screenshots to capture:**
  1. **Home/Dashboard** - Show bounty listings and main navigation
  2. **Create Bounty** - Demonstrate the bounty creation flow
  3. **Bounty Detail** - Show bounty details and accept/apply flow

  5. **Wallet** - Show escrow and payment features
  6. **Profile** - Display user profile and settings

#### 5.5" Display (iPhone 8 Plus)
- **Resolution:** 1242 x 2208 pixels (portrait) or 2208 x 1242 pixels (landscape)
- **Format:** PNG or JPEG
- **Required:** Same set as 6.5" display

#### iPad Pro (12.9-inch) 3rd generation
- **Resolution:** 2048 x 2732 pixels (portrait) or 2732 x 2048 pixels (landscape)
- **Format:** PNG or JPEG
- **Required:** Minimum 3 screenshots if iPad is supported

### Android Screenshots

#### Phone Screenshots
- **Resolution:** Minimum 1080 x 1920 pixels (portrait)
- **Format:** PNG or JPEG (no alpha transparency)
- **Required:** Minimum 2 screenshots, maximum 8
- **Use same scenes as iOS**

#### Tablet Screenshots (Optional but Recommended)
- **Resolution:** Minimum 1536 x 2048 pixels
- **Format:** PNG or JPEG
- **Required if tablet support is claimed**

### Screenshot Best Practices
1. **Remove status bar or use clean demo data:**
   - Use 9:41 AM for time
   - Full battery, full signal
   - No notifications

2. **Show key features:**
   - Trust and security (escrow)
   - Easy creation flow
   - Real-time chat
   - Clean, modern UI

3. **Add text overlays (optional but recommended):**
   - Highlight key features with short, punchy text
   - Use brand colors (emerald theme)
   - Ensure text is readable on all backgrounds

4. **Consistency:**
   - Use same demo data across screenshots
   - Maintain consistent branding
   - Professional, polished appearance

## üé¨ App Preview Video

### Requirements
- **Duration:** 15-30 seconds recommended (max 30 seconds for best engagement)
- **Resolution:** 
  - iOS: 1920 x 1080 pixels (landscape) or 1080 x 1920 pixels (portrait)
  - Android: 1920 x 1080 pixels minimum
- **Format:** MOV, MP4, or M4V
- **File size:** Max 500MB (iOS), 100MB (Android)

### Video Structure (30-second example)

**0-5 seconds:** Hook
- Show app icon and name
- Text: "Need help with a task? Or want to earn extra money?"

**5-15 seconds:** Core Value
- Show bounty creation flow (3-4 seconds)
- Show bounty browsing and acceptance (3-4 seconds)
- Show in-app chat (2-3 seconds)

**15-25 seconds:** Trust & Benefits
- Show escrow payment screen
- Quick cuts of happy users
- Text: "Secure. Fast. Transparent."

**25-30 seconds:** Call to Action
- App icon with download text
- Text: "Download BOUNTY Today"
- Show star ratings (once available)

### Video Best Practices
1. **No audio required** (most users watch muted)
2. **Add captions/text overlays** for key points
3. **Show real UI** (not mockups)
4. **Keep it simple** - focus on 3-4 key features
5. **Use transitions** but keep them subtle
6. **End with clear CTA** (download now)
7. **Test on multiple devices** before submission

### Tools for Creating Video
- **Screen recording:** iOS Screen Recording, Android ADB
- **Editing:** 
  - iMovie (Mac)
  - DaVinci Resolve (Free, professional)
  - Adobe Premiere Rush (Mobile-friendly)
  - CapCut (Simple, fast)
- **Text overlays:** Add in editing software
- **Music:** Optional, but use royalty-free if included

## üìã Pre-Submission Checklist

- [ ] Keywords selected and optimized

- [ ] Build created with `eas build --platform all --profile production`
- [ ] App tested on multiple devices
- [ ] All features working in production build
- [ ] No console.log statements in production code
- [ ] Privacy policy accessible at configured URL
- [ ] Terms of service accessible at configured URL

### Metadata Accuracy
- [ ] Bundle identifier matches app.json
- [ ] Version number is 1.0.0
- [ ] Privacy policy URL added to app.json
- [ ] Terms of service URL added to app.json
- [ ] Age rating determined (17+ recommended for UGC)
- [ ] Support email configured
- [ ] Marketing URL configured (optional)

### Store-Specific Requirements

#### Apple App Store
- [ ] App Store Connect account active
- [ ] Developer Program membership valid ($99/year)
- [ ] Export compliance questionnaire completed
- [ ] IDFA/Tracking disclosure completed
- [ ] TestFlight beta completed (10-20 testers)
- [ ] All reviewer notes written

#### Google Play Store
- [ ] Google Play Console account active ($25 one-time)
- [ ] Content rating questionnaire completed
- [ ] Target audience and content selected
- [ ] Store listing experiments configured (optional)
- [ ] Beta track testing completed
- [ ] All policy compliance checkboxes reviewed

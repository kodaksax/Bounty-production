## Responsive Behavior

### Small Screens (iPhone SE)
- Skeleton cards scale down proportionally
- Same number of items shown
- Touch targets remain 44Ã—44pt minimum

### Large Screens (iPad)
- Skeleton cards maintain proportions
- May show more items to fill viewport
- Grid layout for tablets

## Accessibility Features

### Screen Reader Announcements
```
When loading starts:
  "Loading bounties"

When content loads:
  "5 bounties loaded"

When pagination loads:
  "2 more bounties loaded"
```

### Reduced Motion Support
For users with `prefers-reduced-motion`:
- Pulse animation duration: 2000ms (slower)
- Fade transition: 150ms (faster)
- No scale or complex transforms

## Performance Metrics

### Target Performance
- **Frame Rate:** 60 FPS (16.67ms per frame)
- **Native Driver:** All animations
- **Memory:** No leaks on mount/unmount
- **CPU:** < 5% on mid-range devices

## Edge Cases Handled

1. **Empty Lists:** Show empty state instead of skeletons after timeout
2. **Quick Loads:** Skeletons appear for minimum 200ms to avoid flicker
3. **Error States:** Skeletons fade to error message
4. **Network Retry:** Skeletons reappear on retry
5. **Offline Mode:** Skeletons timeout to offline message
